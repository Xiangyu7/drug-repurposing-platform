DIRECTION A

Cross-Disease Transfer

跨疾病迁移

Discovers drugs approved for Disease X that may treat Disease Y by reversing Y's transcriptomic signature. Exploits shared molecular pathways across conditions.

发现已批准用于疾病 X 的药物可能通过逆转疾病 Y 的转录组特征来治疗 Y。利用不同疾病间的共享分子通路。

DIRECTION B

Origin Reassessment

原适应症重评估

Re-evaluates drugs within their original therapeutic area using updated evidence — safety data, trial outcomes, and new mechanistic insights — to find overlooked candidates.

使用更新的证据（安全性数据、试验结果、新机制发现）在药物原始治疗领域内重新评估，寻找被忽视的候选药物。

Pipeline Deep Dive

管道详解

1

Disease Signature Meta-analysis

dsmeta_signature

Curates and harmonizes disease gene-expression datasets from GEO. Performs differential expression analysis, applies FDR gating, and produces a consensus disease signature (up/down-regulated gene sets).

从 GEO 数据库整理和统一疾病基因表达数据集。执行差异表达分析，应用 FDR 门控，生成一致性疾病特征（上/下调基因集）。

Input输入GEO Series IDs
Output输出DEG Signature (UP/DN)
Key Method核心方法Limma + FDR < 0.05
2

ARCHS4 Expression Extraction

archs4_signature

Queries the ARCHS4 massive gene expression compendium. Auto-generates search keywords from OpenTargets disease synonyms with domain-specific augmentation. Extracts expression signatures when curated GEO datasets are unavailable.

查询 ARCHS4 大规模基因表达数据库。从 OpenTargets 疾病同义词自动生成搜索关键词并添加领域特异性补充。在无可用 GEO 数据集时提取表达特征。

Input输入Disease Keywords
Output输出DEG Signature (UP/DN)
Key Method核心方法ARCHS4 API + Auto-Fallback
3

Signature Reversal Matching

sigreverse

Implements the full CMap 4-stage scoring pipeline: ES (enrichment) → WTCS (weighted directional) → NCS (normalized) → Tau (percentile). Aggregates across cell lines, doses, and time points with robustness filtering (p_reverser ≥ 0.80). Includes dose-response monotonicity checks and toxicity/stress confounder detection.

实现完整的 CMap 四阶段评分管道：ES（富集） → WTCS（加权方向性） → NCS（归一化） → Tau（百分位）。跨细胞系、剂量和时间点聚合，配合鲁棒性过滤（p_reverser ≥ 0.80）。包含剂量-反应单调性检查和毒性/应激混杂检测。

ES→ WTCS→ NCS→ Tau
Input输入DEG Signature
Output输出Ranked Drugs + Tau Scores
Key Method核心方法CMap 4-Stage + Robustness Agg.
4

Knowledge Graph Reasoning

kg_explain

Constructs a drug–gene–disease knowledge graph from OpenTargets, ChEMBL, and FAERS. Generates mechanistic explanations for each drug–disease pair, incorporating genetic associations, safety signals, and clinical trial data.

利用 OpenTargets、ChEMBL 和 FAERS 构建药物–基因–疾病知识图谱。为每个药物–疾病对生成机制解释，整合遗传关联、安全性信号和临床试验数据。

Input输入Drug–Disease Pairs + DEGs
Output输出Evidence Packs + KG Paths
Key Method核心方法Multi-source KG + Path Reasoning
5

LLM + RAG Evidence Synthesis

llm_rag_scoring

Two-stage retrieval pipeline: BM25 broad search on PubMed E-utilities, then semantic reranking via Ollama embeddings. Includes cross-drug contamination detection, negative evidence extraction from ClinicalTrials.gov, and LLM hallucination detection with auto-repair. Outputs a 5-dimensional 100-point score (Evidence 0-30 / Mechanism 0-20 / Translatability 0-20 / Safety 0-20 / Practicality 0-10) plus natural-language rationales.

两阶段检索管道：PubMed E-utilities 的 BM25 广泛搜索，随后通过 Ollama 嵌入进行语义重排序。包含跨药物污染检测、ClinicalTrials.gov 负面证据提取和 LLM 幻觉检测与自动修复。输出 5 维 100 分制评分（证据 0-30 / 机制 0-20 / 可转化性 0-20 / 安全性 0-20 / 实用性 0-10）及自然语言理由。

Input输入Evidence Packs
Output输出5D Scores + Rationales
Key Method核心方法BM25 → Rerank → LLM 5D Scoring
Scoring Engine
评分引擎

V5 Scoring System

V5 评分系统

A multi-dimensional scoring formula that balances mechanistic plausibility with clinical risk signals.

一个多维评分公式，平衡机制合理性与临床风险信号。

// V5 Composite Score
final_score = mechanism_score × risk_decay × (1 + phenotype_boost)

// Risk Decay
safety_penalty  = tanh( log1p(PRR) / 5 × confidence × serious_weight )
trial_penalty   = 0.1 × log1p(safety_stops) + 0.05 × log1p(efficacy_stops)
risk_decay      = (1 - safety_penalty) × (1 - trial_penalty)

// Phenotype Boost
phenotype_boost = avg_pheno_score × log1p(n_phenotypes)

Scientific Rigor

科学严谨性

Drug repurposing predictions are only useful if they are trustworthy. This platform implements multiple layers of statistical validation and bias detection rarely seen in computational drug discovery.

药物重定位预测只有在值得信赖时才有价值。本平台实施了多层统计验证和偏差检测，这在计算药物发现领域极为罕见。

CI

Bootstrap Confidence Intervals

Bootstrap 置信区间

Every drug–disease score is accompanied by uncertainty quantification via 1,000× bootstrap resampling. Predictions are classified into confidence tiers — HIGH (<0.10 CI width), MEDIUM (<0.25), LOW (≥0.25) — so downstream users know exactly how reliable each prediction is.

每个药物–疾病评分都通过 1,000 次 bootstrap 重采样进行不确定性量化。预测按置信度分层 — 高（CI 宽度 <0.10）、中（<0.25）、低（≥0.25）— 让下游用户准确了解每个预测的可靠程度。

ci_lower | ci_upper | ci_width | confidence_tier: HIGH / MEDIUM / LOW
🔒

Data Leakage Auditing

数据泄露审计

Triple-level leakage detection: drug-level (same drug in train/test), disease-level (same disease), and exact pair-level (drug-disease pair overlap). Also detects transductive leakage via shared targets — if test drugs use the same targets that train drugs used. Auto-generated audit reports flag all detected leakage.

三层泄露检测：药物级（训练/测试中的相同药物）、疾病级（相同疾病）和精确对级（药物-疾病对重叠）。还能检测通过共享靶点的转导泄露。自动生成的审计报告标记所有检测到的泄露。

Drug-level ✓ | Disease-level ✓ | Pair-level ✓ | Transductive ✓
A+B

Dual-Route Cross-Validation

双路线交叉验证

Direction A (cross-disease) and Direction B (origin reassessment) run completely independently, then intersect via compare_ab_routes.py. Drugs found by both routes receive a modest confidence boost (2.0, deliberately avoiding winner-takes-all distortion). The overlap set represents the highest-confidence predictions.

方向 A（跨疾病）和方向 B（原适应症重评估）完全独立运行，然后通过 compare_ab_routes.py 交叉验证。两条路线共同发现的药物获得适度置信度提升（2.0，刻意避免赢家通吃偏差）。重叠集代表最高置信度的预测。

Route A+B overlap → highest confidence | A_only | B_only → categorized
⌚

Temporal Split Validation

时序分割验证

Datasets are automatically partitioned by publication year to preserve temporal causality — the model cannot "cheat" by seeing future evidence during training. This integrated temporal benchmark includes a leakage audit within the split itself, preventing information from future publications from contaminating the test set.

数据集按发表年份自动分割以保持时间因果性 — 模型无法通过在训练中看到未来证据来"作弊"。这个集成的时序基准在分割内部包含泄露审计，防止来自未来出版物的信息污染测试集。

Train/test split by year | Future leakage blocked | Temporal causality preserved
