MOTIVATION

问题与方案

药物研发的困境

  • 单个药物平均研发周期超过 10 年
  • 临床试验失败率高达 90%
  • 每个获批药物成本超过 26 亿美元
  • 罕见病和复杂疾病长期缺乏有效治疗手段

药物重定位的方案

  • 利用已有安全性数据，可跳过 I 期临床试验
  • 融合转录组学、知识图谱与大语言模型推理
  • 从疾病特征到候选药物排名的全自动化管线
  • 为每个药物–疾病预测提供可解释的证据包

方向 B

PIPELINE DETAILS

管道详解

1

疾病特征元分析

dsmeta_signature

从 GEO 数据库整理和统一疾病基因表达数据集。执行差异表达分析，应用 FDR 门控（q < 0.05），生成一致性疾病特征（上调/下调基因集）。支持多数据集的 meta-analysis 和加权特征融合。

输入GEO Series IDs
输出DEG Signature (UP/DN)
核心方法Limma + FDR < 0.05
2

ARCHS4 表达特征提取

archs4_signature

查询 ARCHS4 大规模基因表达数据库。从 OpenTargets 疾病同义词自动生成搜索关键词，并添加领域特异性补充（EXTRA_KEYWORDS）。在无可用 GEO 数据集时作为自动回退方案提取表达特征。

输入疾病关键词
输出DEG Signature (UP/DN)
核心方法ARCHS4 API + 自动回退
3

特征逆转匹配

sigreverse

实现完整的 CMap 四阶段评分管道：ES（富集） → WTCS（加权方向性） → NCS（归一化） → Tau（百分位）。跨细胞系、剂量和时间点聚合，配合鲁棒性过滤（p_reverser ≥ 0.80）。包含剂量-反应单调性检查和毒性/应激混杂因子检测。

ES→ WTCS→ NCS→ Tau
输入DEG Signature
输出排名药物 + Tau 分数
核心方法CMap 四阶段 + 鲁棒性聚合
4

知识图谱推理

kg_explain

利用 OpenTargets、ChEMBL 和 FAERS 构建药物–基因–疾病知识图谱。通过 DTPD 四跳路径推理（Drug → Target → Pathway → Disease）生成机制解释，整合遗传关联、安全性信号和临床试验数据。高度连接节点（如激酶家族）施加 hub penalty 降低噪声。

输入药物–疾病对 + DEGs
输出证据包 + KG 路径
核心方法多源 KG + DTPD 路径推理
5

LLM + RAG 证据综合

llm_rag_scoring

两阶段检索管道：PubMed E-utilities 的 BM25 广泛搜索，随后通过 Ollama 嵌入进行语义重排序。包含跨药物污染检测、ClinicalTrials.gov 负面证据提取和 LLM 幻觉检测与自动修复。输出 5 维 100 分制评分（证据 0-30 / 机制 0-20 / 可转化性 0-20 / 安全性 0-20 / 实用性 0-10）及自然语言理由。

输入证据包
输出5D 评分 + 理由
核心方法BM25 → 语义重排 → LLM 5D 评分
SCORING ENGINE

V5 评分系统

一个多维评分公式，平衡机制合理性与临床风险信号，融合多个独立数据源的排名。

// V5 综合评分
final_score = mechanism_score × risk_decay × (1 + phenotype_boost)

// 风险衰减
safety_penalty  = tanh( log1p(PRR) / 5 × confidence × serious_weight )
trial_penalty   = 0.1 × log1p(safety_stops) + 0.05 × log1p(efficacy_stops)
risk_decay      = (1 - safety_penalty) × (1 - trial_penalty)

// 表型增强
phenotype_boost = avg_pheno_score × log1p(n_phenotypes)

VALIDATION

科学严谨性

药物重定位预测只有在值得信赖时才有价值。本平台实施了多层统计验证和偏差检测，这在计算药物发现领域极为罕见。

CI

Bootstrap 置信区间

每个药物–疾病评分都通过 1,000 次 bootstrap 重采样进行不确定性量化。预测按置信度分层 — 高（CI 宽度 <0.10）、中（<0.25）、低（≥0.25）— 让下游用户准确了解每个预测的可靠程度。

ci_lower | ci_upper | ci_width | confidence_tier: HIGH / MEDIUM / LOW
🔒

数据泄露审计

三层泄露检测：药物级（训练/测试中的相同药物）、疾病级（相同疾病）和精确对级（药物-疾病对重叠）。还能检测通过共享靶点的转导泄露 — 防止测试药物使用训练药物的靶点信息。

A+B

双路线交叉验证

方向 A（跨疾病）和方向 B（原适应症重评估）完全独立运行，然后通过 compare_ab_routes.py 交叉验证。两条路线共同发现的药物获得适度置信度提升（2.0，刻意避免赢家通吃偏差），重叠集代表最高置信度预测。

A+B 重叠 → 最高置信度 | A_only | B_only → 分类标记
⌚

时序分割验证

数据集按发表年份自动分割以保持时间因果性 — 模型无法通过在训练中看到未来证据来"作弊"。集成的时序基准在分割内部包含泄露审计，防止来自未来出版物的信息污染测试集。

按年份训练/测试分割 | 未来泄露阻断 | 时间因果性保证

INNOVATION

技术亮点

Δ

FDR 门控差异基因

严格的 FDR 控制（q < 0.05）确保只有统计显著的基因特征进入管道，从源头保证信号质量。

⇄

双路线独立验证

跨疾病和原适应症路线独立运行后交叉验证，两路线重叠的药物代表最高置信度预测。

🔍

可解释人工智能

每个预测都包含结构化证据包：机制解释、PMID 文献支持和知识图谱路径，全程可溯源。

⚖

分子对接就绪

输出包含 SMILES、ChEMBL ID、PDB/AlphaFold 结构 ID 和靶点选择评分 — 可直接用于下游分子对接。

⚡

三级自动回退

GEO 特征提取失败 → 自动切换 ARCHS4 → 再失败回退 OpenTargets-only，无需人工干预即可保证覆盖。

♻

盐型 → 母体解析

通过 ChEMBL 分子层级结构自动将药物盐型解析为母体分子，防止重复条目和评分偏差。

∞

24/7 持续运行器

Ops 运行器持续处理疾病批次，配备状态追踪、自动重启、失败隔离和审计日志。

★

多源知识图谱

将 OpenTargets、ChEMBL、FAERS 和表型数据库整合为统一知识图谱，DTPD 四跳路径推理。

🏭

GEO 自动发现

自动搜索 NCBI GEO 表达数据集，规则检测病例/对照组，质量评分研究 — 新疾病引入不到 5 分钟。

🧬

AlphaFold + PDB 双标注

每个靶点同时标注实验 PDB 和 AlphaFold 预测结构，按可用性分层，配有对接可行性标志和备选靶点。

🎯

动态 Top-N 策略

根据评分分布质量（N50 指标）自动调整候选池大小，三种可调配置：稳定、平衡和召回。告别武断截断。

🔮

幻觉检测与修复

LLM 输出经过模式验证、跨药物污染过滤和自动幻觉检测修复管道，确保证据真实可靠。
