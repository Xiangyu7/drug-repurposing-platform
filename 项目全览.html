2. dsmeta_signature_pipeline

从 GEO 公共数据库下载多组疾病基因表达数据集，做差异表达分析 + 跨研究 meta 分析，输出疾病基因签名。需要预定义 GSE 列表。

1fetch_geo — 下载 GEO 数据 R

根据配置的 GSE ID 列表，从 NCBI GEO 下载表达矩阵和样本信息。

OUT: expr.tsv + pheno.tsv (每个 GSE)

2de_limma — 差异表达分析 R

对每个 GSE 做 limma 差异表达，正则匹配 case/control，PCA 异常值剔除。

OUT: de.tsv (logFC / t-stat / p-value)

3probe_to_gene — 探针/Ensembl → 基因符号 Python Ensembl REST

不同芯片平台用不同探针 ID，统一映射到 HGNC 基因符号。支持 Ensembl ID (ENSG*) 自动转换。

4meta_effects — 跨研究 Meta 分析 R

metafor 随机效应模型合并多个 GSE 的差异表达结果。输出 meta logFC、z 值、FDR、方向一致性。

5rank_aggregate — RRA 排名聚合 Python R

RobustRankAggreg 与 meta 分析互补。集成得分 = 70% meta + 30% RRA。

6-8通路分析 R Python

下载 Reactome/WikiPathways 基因集 → fgsea 富集分析 → Stouffer Z 跨 GSE 合并通路结果。

9make_signature — 生成签名 JSON Python

取 top 300 上调 + top 300 下调基因，生成 JSON 签名文件。

OUT: sigreverse_input.json + up/down_genes.txt

3. archs4_signature_pipeline

从 ARCHS4 H5 文件中按关键词搜索疾病相关 GEO 样本，做 DESeq2 差异表达 + meta 分析。无需预定义 GSE 列表，适合新疾病。

1search_series — 关键词搜索 GEO 系列 Python

在 H5 样本元数据中搜索疾病关键词，自动分配 case/control，按 GEO series 分组。

IN: human_gene_v2.4.h5 + 配置 YAML   OUT: series_candidates.json

2archs4_select — 样本选择 & 表达提取 Python

从 H5 文件直接提取选中样本的基因表达矩阵。每组 min 3 / max 50 样本。

3de_analysis — DESeq2 差异表达 R

ARCHS4 是 count 数据，适合 DESeq2 而非 limma。

4meta_analysis — 跨系列 Meta 分析 Python

DerSimonian-Laird 随机效应模型，要求方向一致性 ≥ 80%。

5make_signature — 生成签名 JSON Python

按 meta_z × OT_score 加权排序，生成与 dsmeta 格式兼容的签名。

OUT: sigreverse_input.json + disease_signature_meta.json

签名源选择：SIG_PRIORITY=dsmeta (默认) 或 archs4，失败自动回退。
关键词来源：OpenTargets 同义词 (自动) + EXTRA_KEYWORDS 字典 (手动补 GEO 友好术语)。
质量门控：total < 30 或 min(up,down) < 10 → 跳过 Cross。30-100 → cap 80 药物。≥100 → cap 200。

4. SigReverse

核心假说：如果某药物能"反转"疾病的基因表达（上调变下调），可能有治疗潜力。基于 LINCS L1000 数据库实现 CMap 方法论。

1加载签名 + 基因→LINCS UUID 映射 LDP3 API

加载 dsmeta/ARCHS4 签名，把基因符号转为 LINCS 数据库中的实体 UUID。

2LINCS L1000 富集查询 LDP3 API

查询 Top 10,000 个与疾病签名最相关的药物-细胞系-剂量-时间签名。

3WTCS 打分 + 药物聚合

签名级 WTCS 打分 → 分类 reverser/mimicker → 药物级聚合：
final = median × p_reverser × log(1+n)/log(9) × cell_line_bonus

4统计检验 + CMap Tau

1000 次置换 → 经验 p 值 → BH FDR。CMap 四阶段：ES → WTCS → NCS → Tau (-100~+100)。Tau < -90 = 强反转。

5剂量反应 + QC

Spearman 相关分析剂量依赖性 + 毒性假阳性检测。

OUT: drug_reversal_rank.csv (候选药反转分数排名)

6. LLM+RAG 证据工程

用 PubMed 文献检索 + 本地 LLM 结构化提取，为候选药生成文献证据，多维打分 + 门控筛选。Step 6-9 对 Direction A 和 B 各跑一遍。

6PubMed RAG + LLM 证据提取 PubMed Ollama qwen2.5:7b

三阶段检索：PubMed 多路检索 → BM25 Top 80 → 嵌入语义重排 Top 30 → LLM 结构化提取
三层幻觉检测：PMID 一致性 + 药物锚定 + 机制锚定

OUT: dossiers/*.json + step6_rank_v2.csv

75 维打分 + GO/MAYBE/NO-GO 门控

证据强度(30) + 机制合理性(20) + 可转化性(20) + 安全性(20) + 可行性(10) = 满分 100
硬门控：benefit<2 / 文献<3 / 危害>50% → NO-GO
软门控：≥60→GO / 40-60→MAYBE / <40→NO-GO

OUT: step7_gating_decision.csv + step7_hypothesis_cards.md

8候选打包 + Excel 报告

移除 NO-GO → Top-K 打包 → Excel 工作簿（Shortlist sheet + 每药 Sheet + 靶点结构表）。

OUT: step8_fusion_rank_report.xlsx + step8_shortlist_topK.csv

kg_explain 产出两个 bridge 文件，分裂成两条独立路线，回答不同的科学问题。

Direction A: 跨疾病迁移 "其他疾病的药能否迁移到目标疾病？"

每个药物的全局最优疾病得分
探索性 (Exploration)
高风险 / 高回报 / 高新颖性

输入: bridge_repurpose_cross.csv
输出: step{6-9}_repurpose_cross/
Direction B: 原疾病重评估 "失败的药是否因选错终点/人群而失败？"

只看目标疾病内的得分
利用性 (Exploitation)
低风险 / 中等回报 / 更多数据

输入: bridge_origin_reassess.csv
输出: step{6-9}_origin_reassess/

8. 打分体系

分数沿管线串行传递，最终在 Step 8 汇合。

层级模块核心公式方向
L1 SigReverse WTCS 反转打分 → 聚合 + 多细胞系 bonus 更负=更好
L2 kg_explain V5 mechanism × exp(-safety - trial) × (1 + pheno)
safety: tanh(log1p(PRR)/5 × conf × wt)   trial: 0.1×log1p(safe) + 0.05×log1p(eff)   pheno: avg_score × log1p(n)		 更高=更好
L3 LLM 5 维 证据(30) + 机制(20) + 转化(20) + 安全(20) + 可行(10) = 0-100 → GO/MAYBE/NO-GO 更高=更好
L4 Step 8 rank_key
(最终融合点)		 文献量 + topic_match + LLM_score×0.35 + novelty×15~19.5 - uncertainty - harm - neg_trials - blacklist 更高=更好
设计要点：LLM_score 权重压低到 ×0.35（防止偏向"好讲故事"的药）。组合药按成分数归一化。Bootstrap CI 1000x 量化不确定性。A/B 交叉验证是额外独立性检验。
